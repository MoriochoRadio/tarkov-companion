# 추천 빌드 데이터 (`public/data/builds.json`) 작성 가이드

모딩 탭의 "추천 빌드" 카드가 읽는 정적 데이터. 직접 수정·추가할 수 있다.
가격·이름·아이콘은 사이트가 tarkov.dev에서 실시간으로 입히므로 **여기에는 id와
설명만** 넣으면 된다.

## 형식

```jsonc
{
  "updated": "2026-06-12",        // 마지막 수정일 (YYYY-MM-DD)
  "builds": [
    {
      "id": "mosin-ll1-pu",        // 고유 슬러그 (소문자-하이픈). 무기-티어-특징 권장
      "weapon": "5ae08f0a5acfc408fb1398a1",  // 무기 아이템 id (tarkov.dev)
      "name": "모신 PU 국밥 저격",  // 카드 제목 (한국어)
      "category": "sniper",        // ar | smg | dmr | shotgun | sniper
      "tier": 1,                   // 트레이더 레벨 티어 1~4 — "이 레벨이면 살 수 있다"
      "parts": [                   // 부품 아이템 id 배열 (순서는 표시 순서)
        "5b3f7bf05acfc433000ecf6b",
        "5b3f7c005acfc4704b4a1de8",
        "5b3f7c1c5acfc40dc5296b1d"
      ],
      "desc": "왜 가성비인지 한 줄 설명",
      "source": "https://..."      // (선택) 참고 영상/위키 링크
    }
  ]
}
```

## 부품 id 찾는 법

```
node scripts/explore-weapon.mjs <무기id> --tier 2
```

무기의 슬롯별 장착 가능 부품을 "그 티어에서 현금 구매 가능한 것만" 추려서
id·가격·에르고·반동과 함께 출력한다. `sub:N`이 붙은 부품(핸드가드·마운트 등)은
그 부품 id로 다시 실행하면 안쪽 슬롯(포어그립·스코프)을 볼 수 있다.
무기 id는 모딩 탭 "부품 직접 탐색"이나 tarkov.dev 아이템 페이지 URL에서 확인.

## 장착 규칙 (검증기가 보는 것)

부품은 다음 중 하나의 슬롯에 들어가면 통과:

1. 무기 자체의 슬롯 (개머리판, 탄창 등)
2. 빌드에 포함된 **다른 부품**의 슬롯 — 마운트 위 도트처럼 체인이 되면
   중간 부품(마운트)도 반드시 `parts`에 넣을 것
3. 무기 **기본 프리셋** 부품의 슬롯 — 기본 핸드가드에 포어그립을 다는 경우
   핸드가드는 안 넣어도 됨

기본 장착돼 나오는 부품(기본 탄창 등)은 넣지 않는다 — 비용 합산이 이중으로
잡힌다.

## 수정 후 반드시

```
node scripts/validate-builds.mjs
```

- `✗ 장착 불가` — id를 잘못 골랐거나 체인 중간 부품이 빠짐. 커밋 금지
- `⚠ LL{n} 현금 구매 불가 / 퀘스트 해금` — 빌드는 유효하지만 그 티어에
  트레이더 현금으로 못 사는 부품 (사이트는 플리 시세로 폴백). 의도한 경우
  `desc`에 명시할 것 (예: "예거 퀘스트 해금")

전부 통과(`실패 0`)한 상태로만 커밋한다. 게임 패치로 부품·세율이 바뀔 수
있으니 와이프 후에는 한 번씩 다시 돌려보는 것을 권장.
